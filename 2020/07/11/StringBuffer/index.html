    StringBuffer

    StringBuffer是字符串缓冲区，也可以说是一个容器.
    与String一样，StringBuffer也不可以被继承->public final class StringBuffer

    作为容器，它具有存储，删除，获取，修改的功能.而且长度是可以变化的，可以操作多个数据类型，最终可通过toString()方法变成字符串.

    1.存储

    StringBuffer append():将指定数据作为参数添加到已有的数据尾处；
    StrngBuffer insert(index,data):在指定位置插入数据；

    2.删除

    StringBuffer delete(int start,int end),删除缓冲区中的数据，包含start,不含end;
    StringBuffer deleteCharAt(index),删除指定位置的字符；

    3.获取

    char charAt(int index),获取字符；
    int indexOf(String str),获取字符串的位置；
    int lastIndexOf(String str),获取字符串最后一次位置；
    int length()，获取长度；
    String substring(int start,int end),获取子串；

    4.修改

    StringBuffer replace(int start,int end,Stirng str),

    void setCharAt(int index,char ch),修改一处的字符，不返回任何值

    5.反转

    StringBuffer reverse()，将StringBuffer中的内容反转；

    6.转换

    void getChars(int srcBegin, int srcEnd, char[] dst, int dstBegin),把原串的指定起点终点的子串转化为一个个字符并放入目标字符数组dst，以dstBegin为起点.

    StringBuilder类

    一个可变的字符序列，提供一个与StringBuffer兼容的API,但不保证同步，被设计用作StringBuffer的一个简易替换，在JDK1.5版本之后出现.

    StringBuffer是线程同步，例如在执行获取操作时不能进行其他操作，多线程时使用安全；
    StringBuilder是线程不同步，用于多个线程是不安全的，建议单线程使用.

    方法与StringBuffer的方法几乎一模一样.
