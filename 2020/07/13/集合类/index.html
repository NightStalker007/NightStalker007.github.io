    1.简介

    面向对象语言对事物的体现都是以对象的形式，所以为了方便对多个对象的操作，就对对象进行存储，集合就是存储对象最常用的一种方式.

    数组虽然也可以存储对象，但长度是固定的；集合长度是可变的。数组中可以存储基本数据类型，但是集合只能存储对象，但是可以存储不同类型的对象.

    每一个容器对数据的存储方式都有所不同，这个存储方式称之为数据结构.

    2.List

    List: 元素是有序的，元素可以重复，因为该集合体系有索引；
    –ArrayList:底层的数据结构使用的是数组结构；特点：查询速度很快，但是增删稍慢；
    –LinkedList:底层使用的链表数据结构；增删速度快，查询慢；
    –Vector:底层是数组数据结构，线程同步，被ArrayList替代.

    Set: 元素是无序的，元素不可以重复，集合中无索引.

    添加

    add(index,element),特定位置插入element;
    addAll(index,Collection),特定位置加入特定Collection中所有元素；

    删除

    remove(index),删除特定位置的元素；

    修改

    set(index,element),修改特定位置的元素；

    查找

    get(index),位置找元素；
    subList(fromindex,toindex),返回子串，要头不要尾；
    listIterator(),读取.

    要注意，List中的元素判断是否相同使用equals方法.

    很多方法与下面ArrayList相似，这里演示listiterator();

    3.ArrayList

    ArrayList是一个数组队列，相当于动态数组。与Java中的数组相比，它的容量能动态增长。它继承于AbstractList，实现了List，RandomAccess，Cloneable，java.io.Serializable这些接口。

    常用方法

    添加

    boolean add(E e),添加元素至末尾；
    void add(int index,E element),添加特定元素至特定位置；

    获取

    int size(),得到长度；
    E get(int index),得到指定位置的元素；

    删除

    void clear(),清空列表；
    boolean remove(Object o),移除特定元素；
    E remove(int index),移除指定位置的元素；

    判断

    boolean contains(Object o),当列表含有特定元素时返回true;
    boolean isEmpty(),当列表为空时返回true;

    其他

    boolean retainAll(Collection c),取二者交集，两者都含有的部分；
    boolean removeAll(Collection c),移除此collection中也包含在指定collection中的所有元素.

    LinkedList

    LinkedList与前面几种结构有许多类似的方法，这里演示特有方法.

    LinkedList addFirst(obj),在表头插入元素；
    LinkedList addLast(obj),在表尾插入元素；
    obj getFirst(),返回第一个元素；
    obj getLast(),返回最后一个元素；
    int size(),得到长度(大小)；
    Obj removeFirst(),删除首个元素；
    Obj removeLast(),删除最后一个元素；
    JDK1.6后；

    offerFirst(),offerLast()具有插入功能；
    peekFirst(),peekLast()具有获取元素但不删除功能；

    public E pollFirst(),获取并移除此列表第一个元素，若列表为空，返回null;
    public E pollLast(),获取并移除此列表最后一个元素，若列表为空，返回null；

    队列queue先进先出，栈stack先进后出；

    栈只需改动先出顺序

    HashSet

    底层数据结构是哈希表；

    由于唯一性，只会出现一个java03;

    HashSet存入会比较地址，两个对象一样，不再存入！倘若复写hashCode()方法，使得每一个存入对象的地址相同，则在存入新对象时，会比较地址，地址一样，倘若不是同一对象，则会在同一地址下新开辟空间存储新的对象.这样就保证了Hashset元素的唯一性.简单说，如果元素的HashCode值相同，就会调用equals，否则不需要判断直接加入.

    LeetCode 824

    给定一个由空格分割单词的句子 S。每个单词只包含大写或小写字母。

    我们要将句子转换为 “Goat Latin”（一种类似于 猪拉丁文 - Pig Latin 的虚构语言）。

    山羊拉丁文的规则如下：

    如果单词以元音开头（a, e, i, o, u），在单词后添加”ma”。
    例如，单词”apple”变为”applema”。

    如果单词以辅音字母开头（即非元音字母），移除第一个字符并将它放到末尾，之后再添加”ma”。
    例如，单词”goat”变为”oatgma”。

    根据单词在句子中的索引，在单词最后添加与索引相同数量的字母’a’，索引从1开始。
    例如，在第一个单词后添加”a”，在第二个单词后添加”aa”，以此类推。
    返回将 S 转换为山羊拉丁文后的句子。
    链接：https://leetcode-cn.com/problems/goat-latin

    TreeSet

    TreeSet可以对集合中的元素进行排序；使用TreeSet需要指定排序的准则；

    int compareTo(T o),比较此对象与指定对象（o）的顺序，小于，等于，大于分别返回负整数，0，正整数;

    让类具有比较性需要让类实现Comparable接口，复写compareTo方法；
    实例，将人按年龄优先，姓名字典序升序排列；

    我们可以像上例那样给要比较的对象所属的类自定义compareTo的方法，再将其加入TreeSet。而当元素本身不具备比较性，或者具备的比较性不是所需要的，也可以通过比较器Comparator，来传递一个比较方法的对象给TreeSet，在对象里定义compare方法，从而改变TreeSet排序的准则.

    需要注意的是，倘若两种排序准则都存在(Comparable的元素，Comparator的比较器)，以比较器为主.
    而且比较器方法相对常用.
